# Macros and Constants

How the enum/constant system works, the key macros agents will encounter, and
where to find specific IDs.

---

## The Enum System (`macros/enum.asm`)

All numeric constants are generated by two families of macros.

### `const` — sequential integer constants

```asm
const_def                   ; reset const_value to 0
const HEATH_VILLAGE         ; HEATH_VILLAGE EQU 0, const_value → 1
const ROUTE_69              ; ROUTE_69 EQU 1, const_value → 2
const ROUTE_70              ; ROUTE_70 EQU 2, const_value → 3
```

`shift_const NAME` assigns `NAME EQU (1 << const_value)` then increments —
used for bitmask constants.

### `enum` — arbitrary-start sequential constants

```asm
enum_start 5                ; start from 5
enum ITEM_A                 ; ITEM_A = 5
enum ITEM_B                 ; ITEM_B = 6
enum_set 10                 ; jump to 10
enum ITEM_C                 ; ITEM_C = 10
```

`enum_start` with no argument defaults to 0. `enum_start value, -1` counts down.

### Why this matters for agents

When you see `const MOVE_POUND` in a constants file, it means `MOVE_POUND`
equals the count of `const` calls since the last `const_def`. To find a
specific constant's numeric value, count the entries or search for
`const_value` assignments and `const_def` resets.

## Key Macro Files

### `macros/rst.asm` — Far Calls

These are the most-used macros in the codebase:

```asm
farcall SomeBankedFunction  ; call a function in any ROM bank
                            ; equivalent to: push bank, call, pop bank

callab SomeBankedFunction   ; like farcall but preserves more registers

predef SomePredefined       ; call via the predef table in data/predefs.asm
                            ; safe to use from any context
```

Use `farcall` when calling code in banked ROM from bank 0 (`home/`).
Use `predef` when calling across banks from banked code.

### `macros/map.asm` — Map Data Helpers

```asm
warp_def y, x, dest_warp, DEST_MAP
coord_event y, x, flag_id, ScriptPointer
signpost y, x, SIGNPOST_TYPE, TextOrScriptPointer
person_event SPRITE, y, x, MOVEMENT, PALETTE, TYPE, TEXT, FN, FLAG
```

See [maps-and-events.md](maps-and-events.md) for full field documentation.

### `macros/text.asm` — Dialogue Strings

```asm
TextLabel:
    text "First line"
    line "Second line"
    cont "After button press"
    para "New paragraph"
    done
```

`text` starts a block; `line` continues on the next line in the same box;
`cont` waits for a button press then continues; `para` clears the text box;
`done` ends the block.

### `macros/movement.asm` — NPC Movement Scripts

Used in map files to define NPC walking paths:

```asm
MovementScript:
    step_right  2       ; walk right 2 tiles
    step_up     1       ; walk up 1 tile
    turn_head DOWN      ; turn to face down
    end_movement        ; done
```

### `macros/basestats.asm` — Pokémon Data Builder

Helper that fills out a base stats struct entry from named fields. See
[data-formats.md](data-formats.md) for the full struct layout.

### `macros/pals.asm` — Palette Colors

```asm
RGB 31, 31, 31      ; white (5-bit R, G, B)
RGB  0,  0,  0      ; black
```

Expands to a 2-byte little-endian GBC palette entry.

## How `includes.asm` Works

Nearly every `.asm` file begins with:

```asm
INCLUDE "includes.asm"
```

`includes.asm` pulls in both master files:

```asm
INCLUDE "constants.asm"
INCLUDE "macros.asm"
```

`constants.asm` in turn includes all 38 files in `constants/`, and `macros.asm`
includes all 21 files in `macros/`. This means every symbol defined anywhere in
`constants/` or `macros/` is available in every `.asm` file.

## Finding a Constant's Value

1. **Search by name prefix**: constants follow `CATEGORY_NAME` naming.
   - Species: `SPECIES_*` → `constants/pokemon_constants.asm`
   - Moves: `MOVE_*` → `constants/move_constants.asm`
   - Items: `ITEM_*` → `constants/item_constants.asm`
   - Maps: `MAP_*` → `constants/map_constants.asm`
   - Events: `EVENT_*` → `constants/event_flags.asm`
   - Music: `MUSIC_*` → `constants/music_constants.asm`
   - Sprites: `SPRITE_*` → `constants/sprite_constants.asm`

2. **For numeric value**: count `const` entries from the preceding `const_def`,
   or run a build and inspect the generated `.sym` file:
   ```
   grep "MOVE_SURF" pokeprism_nodebug.sym
   ```

## Common Bit-Packing Macros

```asm
dn a, b         ; pack two 4-bit nibbles into one byte: (a << 4) | b
                ; used for egg groups, DVs, etc.

dbw byte, word  ; emit a 1-byte value followed by a 2-byte value
dwb word, byte  ; emit a 2-byte value followed by a 1-byte value
dba addr        ; emit BANK(addr), LOW(addr), HIGH(addr) — 3-byte far pointer
```

## Predef System

`data/predefs.asm` contains a table of function pointers that can be called
by index from any bank using the `predef` macro. This avoids hardcoding bank
numbers in callers.

To add a new predef:
1. Add an entry to `data/predefs.asm`: `add_predef MyFunction`
2. Add the constant to the predef enum in `constants/` (if one exists) or use
   the generated index directly.
3. Call it: `predef MyFunction` (the macro resolves the index at assembly time).

## Debug Mode

The debug build (`make prism`, which uses `-DDEBUG_MODE`) enables:
- A debug menu accessible in-game
- Extra crash information
- Emulator detection routines

Conditionally-compiled code uses:

```asm
IF DEF(DEBUG_MODE)
    ; debug-only code
ENDC
```

The nodebug build (`make nodebug`) strips all of this out.
